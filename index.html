Credit Eligibility BRE Simulator
Simulator only. Use these three controls to demonstrate the Credit Eligibility BRE flow. Recommendation Tab > Extra Input Fields Sub-Tab and Recommendation Tab > Recommendations & Deviations Sub-Tab. Deviation Description is read-only, Remarks are editable, and stale-result checks plus Forward blockers are handled inside the normal page flow.
1
BRE Eligibility Gate
Called when the deal/application opens. This decides whether the Credit Eligibility BRE flow is applicable.
v
2
Extra Input Fields
Captures BRE inputs not available in LOS. Save shows a simple missing-fields popup; Run blocks unsaved inputs or shows BRE-returned validation errors.
v
3
BRE Result
Passed allows clean recommendations. Deviated shows read-only Deviation Description and requires Remarks. Failed shows read-only Failed Rules and blocks Forward.
Default state has mandatory inputs filled and saved. If extra fields are edited and not saved, Run Credit Eligibility BRE shows the Extra fields not saved popup.
